Best Practice för Scratch-projekt

Dessa riktlinjer hjälper dig bygga stabila, tydliga och lättfelsökta Scratch-projekt. Perfekt när du bygger större spel som plattformsspelet i kursen.

1. Döp allt tydligt

  • Sprites: Spelare, Fiende1, RörligPlattform
  • Variabler: Liv, YHastighet, Nivå
Undvik namn som Sprite1, Test eller Variabel1.

2. En sprite = en uppgift

Varje sprite ska ha ett tydligt ansvar.

  • Spelare = rörelse, hopp, animation
  • Fiende = patrullera, skada spelaren
  • Plattform = röra sig
  • Flagga = byta nivå

3. Använd meddelanden för att styra spelet

Meddelanden är Scratchs sätt att skapa struktur.

  • Starta spel
  • Nästa nivå
  • Game Over
  • Spelare dog

4. Separera logik från grafik

Ha olika script för:

  • rörelse
  • kollision
  • animation
  • effekter

5. Använd variabler smart

Skapa bara variabler som behövs och döp dem tydligt.

6. Använd listor för avancerade spel

Listor är perfekta för nivåer, highscore och sparsystem.

7. Testa ofta – bygg i små steg

Lägg till en funktion → testa → fixa → gå vidare.

8. Använd färgkollisioner med omsorg

  • Starka färger
  • Inga gradienter
  • Inga tunna linjer

9. Håll koden ren och grupperad

  • Använd kommentarer
  • Gruppera block som hör ihop
  • Dela upp långa script

10. Optimera prestanda

  • Undvik onödiga “för alltid”-loopar
  • Ta bort kloner när de inte behövs
  • Använd vänta 0 sekunder för att avlasta

11. Planera innan du bygger

En enkel plan sparar mycket tid.

12. Spara ofta och använd versioner

Döp versioner: Plattformsspel v1, v2, v3.
